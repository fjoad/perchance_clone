# Local VN Current-Space Research Plan

Status: active research plan
Date: May 23, 2026

## Research Question

Given:

- fully local/offline runtime
- zero recurring budget
- RTX 3080 12GB plus system RAM
- unrestricted/adult-capable roleplay requirement
- Perchance-quality-or-better interactive fiction text
- visual novel/storybook image panels
- text streaming quickly and images appearing within a tolerable delay

What currently available text models, image models, runtimes, and orchestration patterns are realistic for this project?

This research is not asking whether offline is desirable. Offline is a constraint.

## Success Criteria

The output of the deep dive should answer:

- Which current text model families are worth testing for Perchance-quality VN prose?
- Which text models are realistic for co-residency with image generation?
- Which text models are realistic only under GPU swapping?
- Which large models are worth treating as quality ceilings?
- Which image model families are currently best for local anime/VN panels?
- Which image backends are most likely to solve speed/VRAM/reload issues?
- Which stack combinations should be tested first on this exact hardware?
- What experiments should be built next, with saved text/images/timings?

## Text Research Scope

### Model Families

Research current local/open-weight models by practical tier:

- 2B-4B: helper/co-resident candidates
- 7B-9B: co-resident main-text candidates
- 10B-14B: likely main-text quality/speed sweet spot
- 20B-35B: quality-ceiling experiments
- MoE models: active-parameter advantage versus total memory cost

### Roleplay/Creative Quality

Prioritize sources that discuss or demonstrate:

- roleplay writing
- interactive fiction
- character adherence
- long-form narrative prose
- dialogue/action balance
- uncensored or abliterated behavior
- SillyTavern/Kobold/llama.cpp usage

### Runtime Compatibility

For each serious candidate, note whether it is available as:

- GGUF for llama.cpp/Ollama/KoboldCpp
- EXL2/GPTQ for TabbyAPI/ExLlamaV2
- Ollama tag
- Hugging Face repo with clear quant files

### Text Outputs To Decide Later

The research should lead to a text benchmark harness that tests:

- first-token latency
- total reply time
- tokens/sec
- prompt-following
- prose quality
- character voice
- use of protagonist profile
- multi-turn continuity
- refusal/filters

## Image Research Scope

### Model Families

Research current local image candidates:

- SDXL anime/illustration models
- Illustrious / NoobAI / WAI-style SDXL descendants
- Pony-family models
- SD 1.5 anime fallbacks
- FLUX and newer image families only if realistic on 12GB
- distilled/few-step SDXL variants such as Lightning, Hyper, Turbo, LCM

### Backend/Runtime

Research:

- current Diffusers baseline
- A1111
- Forge
- ComfyUI
- ComfyUI Dynamic VRAM
- GGUF/quantized image model workflows
- VAE tiling/slicing, xformers, attention optimizations, model offload/reload behavior

### Image Outputs To Decide Later

The image benchmark should save:

- prompt
- negative prompt
- settings
- seed
- final image
- stage image if two-pass
- generation time
- load/unload time
- VRAM before/after

## Orchestration Research Scope

Research viable local architecture patterns:

- text-first, image-after
- text streams while image waits
- image prompt generated by main model
- image prompt extracted by helper model
- image prompt composed by rules/templates
- text/image one-GPU swapping
- co-resident smaller models
- backend-managed dynamic VRAM/offload
- external local servers versus app-owned model objects

The goal is not to pick the prettiest architecture. The goal is to identify what should be tested.

## Evidence Standard

Use:

- official model/runtime docs for capabilities and sizes
- Hugging Face model cards for quants and intended usage
- Ollama tags for local pull sizes
- SillyTavern/Kobold/ComfyUI/Forge docs for runtime patterns
- community guides only when clearly labeled as community signal

Avoid:

- unsupported hype
- one-off benchmark claims without source
- treating leaderboard strength as roleplay quality
- assuming model size predicts VN quality
